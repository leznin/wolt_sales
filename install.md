# Руководство по установке на сервер Ubuntu

Данный документ предоставляет пошаговые инструкции по установке и запуску этого проекта на сервере Ubuntu.

## Предварительные требования

- Ubuntu Server 20.04 LTS или новее
- Доступ к root или права sudo
- Git
- Node.js (версия 14 или новее)
- NPM (версия 6 или новее)
- MongoDB (опционально, если ваш проект использует базу данных)

## Шаг 1: Обновление системы

Откройте терминал на сервере Ubuntu. Вы увидите черное окно с текстовым курсором. Введите следующие команды (нажимайте Enter после каждой строки):

```bash
sudo apt update
```

Система запросит ваш пароль. Введите его (символы пароля не будут отображаться на экране) и нажмите Enter.

Затем введите:

```bash
sudo apt upgrade -y
```

Эта команда обновит все установленные пакеты системы. Флаг `-y` автоматически отвечает "да" на все вопросы.

## Шаг 2: Установка Node.js и NPM

Введите следующие команды в терминал:

```bash
curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
```

Эта команда добавляет репозиторий Node.js в ваш список источников программного обеспечения.

Затем установите Node.js:

```bash
sudo apt install -y nodejs
```

Чтобы проверить, что установка прошла успешно, введите:

```bash
node -v
```

Должна отобразиться версия Node.js (например, v16.x.x).

Проверьте версию NPM:

```bash
npm -v
```

Должна отобразиться версия NPM (например, 6.x.x или 8.x.x).

## Шаг 3: Установка Git (если еще не установлен)

```bash
sudo apt install git -y
```

Чтобы проверить, что Git установлен, введите:

```bash
git --version
```

Должна отобразиться версия Git.

## Шаг 4: Клонирование репозитория

Введите команду для клонирования проекта (замените [URL_ВАШЕГО_РЕПОЗИТОРИЯ] на актуальный URL):

```bash
git clone [URL_ВАШЕГО_РЕПОЗИТОРИЯ]
```

Пример: `git clone https://github.com/username/project.git`

После завершения клонирования перейдите в директорию проекта:

```bash
cd [НАЗВАНИЕ_ДИРЕКТОРИИ_ПРОЕКТА]
```

Пример: `cd project`

## Шаг 5: Установка зависимостей

Находясь в директории проекта, выполните:

```bash
npm install
```

Эта команда установит все необходимые зависимости, указанные в файле package.json. Процесс может занять несколько минут.

## Шаг 6: Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```bash
cp .env.example .env
```

Если файл `.env.example` отсутствует, создайте .env вручную:

```bash
nano .env
```

Откроется текстовый редактор. Введите необходимые переменные окружения, например:

```
PORT=3000
NODE_ENV=production
DATABASE_URL=mongodb://localhost:27017/mydatabase
```

Для сохранения файла и выхода из редактора nano нажмите:
1. Ctrl + O (сохранить)
2. Enter (подтвердить имя файла)
3. Ctrl + X (выйти)

## Шаг 7: Сборка приложения

```bash
npm run build
```

Эта команда соберет ваше приложение для продакшн использования.

## Шаг 8: Настройка менеджера процессов (PM2)

PM2 - это менеджер процессов для приложений Node.js, который обеспечивает работу вашего приложения даже после перезапуска сервера.

```bash
sudo npm install -g pm2
```

Запустите ваше приложение с помощью PM2:

```bash
pm2 start npm --name "app" -- start
```

Здесь "app" - это имя, которое PM2 будет использовать для отслеживания вашего приложения. Вы можете заменить его на любое другое имя.

Сохраните текущий список процессов:

```bash
pm2 save
```

Настройте автозапуск PM2 при старте системы:

```bash
pm2 startup
```

PM2 может вывести дополнительную команду, которую нужно будет выполнить. Скопируйте и выполните эту команду, если она появится.

## Шаг 9: Настройка Nginx в качестве обратного прокси

Установите Nginx:

```bash
sudo apt install nginx -y
```

Создайте новый файл конфигурации Nginx:

```bash
sudo nano /etc/nginx/sites-available/myapp
```

Откроется текстовый редактор. Вставьте следующую конфигурацию (измените по необходимости):

```nginx
server {
    listen 80;
    server_name ваш-домен.com www.ваш-домен.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Замените `ваш-домен.com` на ваш реальный домен или IP-адрес сервера.
Замените `3000` на порт, на котором работает ваше приложение (указанный в .env или в коде).

Сохраните файл: 
1. Ctrl + O
2. Enter
3. Ctrl + X

Создайте символическую ссылку на файл конфигурации:

```bash
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
```

Проверьте правильность конфигурации Nginx:

```bash
sudo nginx -t
```

Если проверка прошла успешно, перезапустите Nginx:

```bash
sudo systemctl restart nginx
```

## Шаг 10: Настройка SSL с Let's Encrypt (опционально)

Если вы хотите настроить безопасное соединение HTTPS:

```bash
sudo apt install certbot python3-certbot-nginx -y
```

Запустите Certbot для получения SSL-сертификата:

```bash
sudo certbot --nginx -d ваш-домен.com -d www.ваш-домен.com
```

Замените `ваш-домен.com` на ваш реальный домен.

Certbot задаст несколько вопросов:
1. Ваш email (для уведомлений о продлении сертификата)
2. Согласие с условиями обслуживания (введите A, чтобы согласиться)
3. Хотите ли вы делиться своим email (введите Y или N)
4. Перенаправлять ли HTTP трафик на HTTPS (рекомендуется выбрать 2 для перенаправления)

## Шаг 11: Настройка файервола

```bash
sudo ufw allow 'Nginx Full'
```

Эта команда разрешает входящие соединения для Nginx на портах 80 (HTTP) и 443 (HTTPS).

```bash
sudo ufw allow ssh
```

Эта команда разрешает SSH-соединения, чтобы вы не потеряли доступ к серверу.

```bash
sudo ufw enable
```

Эта команда включает файервол. Система спросит подтверждение - введите y и нажмите Enter.

## Мониторинг и обслуживание

### Просмотр логов приложения

```bash
pm2 logs app
```

Для выхода из просмотра логов нажмите Ctrl + C.

### Перезапуск приложения

```bash
pm2 restart app
```

### Обновление приложения

```bash
cd [НАЗВАНИЕ_ДИРЕКТОРИИ_ПРОЕКТА]
git pull
npm install
npm run build
pm2 restart app
```

### Просмотр списка запущенных приложений

```bash
pm2 list
```

### Мониторинг использования ресурсов

```bash
pm2 monit
```

Для выхода из монитора нажмите Ctrl + C.

## Устранение неполадок

- **Проблемы с правами доступа**: Если вы сталкиваетесь с ошибками доступа, убедитесь, что ваш пользователь имеет необходимые права. Можно попробовать команду `sudo` перед проблемной командой.

- **Проверка логов приложения**: `pm2 logs app`

- **Проверка логов Nginx**:
  ```bash
  sudo tail -f /var/log/nginx/error.log
  ```
  Для выхода из просмотра логов нажмите Ctrl + C.

- **Проверка статуса Nginx**:
  ```bash
  sudo systemctl status nginx
  ```
  Для выхода из просмотра статуса нажмите q.

- **Перезапуск Nginx**:
  ```bash
  sudo systemctl restart nginx
  ```

- **Проверка переменных окружения**: Убедитесь, что все переменные окружения в файле .env установлены правильно.

- **Проверка доступности порта**:
  ```bash
  sudo netstat -tuln | grep LISTEN
  ```
  Эта команда покажет все порты, находящиеся в состоянии прослушивания.

За дополнительной помощью обращайтесь к сопровождающему проекта.
